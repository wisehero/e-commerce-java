# 상품 판매 정책

작성: 2026-06-13 · 상태: 구현 완료(이번 이터레이션 범위)

이 문서는 상품이 고객에게 어떻게 노출되고, 어떤 기준으로 판매 가능하다고 판단되는지를 정의한다.
구현 구조와 기술적 근거는 [`product-domain-spec.md`](./product-domain-spec.md)를 따른다.

## 1. 목적

상품 판매 정책은 상품, SKU, 가격, 재고, 판매 상태의 비즈니스 의미를 정의한다.

이 정책은 다음 질문에 답한다.

- 상품과 SKU는 각각 무엇을 의미하는가?
- 고객에게 노출되는 상품은 어떤 기준으로 결정되는가?
- 고객이 구매할 수 있는 상품은 어떤 기준으로 결정되는가?
- 정가와 판매가는 각각 어떤 의미를 갖는가?
- 품절은 상품 상태인가, 재고에서 파생되는 표시인가?

## 2. 핵심 용어

| 용어 | 의미 |
|---|---|
| 상품 | 고객에게 보이는 카탈로그 단위 |
| SKU | 실제 판매되는 옵션 조합 단위 |
| 옵션 | 색상, 사이즈처럼 SKU를 구분하는 선택값 |
| 정가 | 할인 전 기준 가격 |
| 판매가 | 고객이 실제 주문·결제에 사용하는 가격 |
| 판매 가능 재고 | 현재 commerce 시스템에서 고객에게 판매할 수 있다고 판단하는 수량 |
| 판매중 | 고객에게 노출되고 구매 가능한 상품 상태 |
| 판매중지 | 일시적으로 고객에게 노출·판매하지 않는 상품 상태 |
| 단종 | 영구적으로 판매를 종료한 상품 상태 |
| 품절 | 판매 가능한 SKU 재고가 없는 상태에서 파생되는 표시 |

## 3. 상품과 SKU 정책

상품은 고객에게 보이는 카탈로그 단위이고, SKU는 고객이 실제로 구매하는 판매 단위다.

- 하나의 상품은 하나 이상의 SKU를 가질 수 있다.
- 상품 등록 시 최소 하나 이상의 SKU가 필요하다.
- 상품명은 비어 있을 수 없고 100자를 초과할 수 없다.
- 카테고리 ID는 필수다. 상품은 반드시 최하위(소분류) 카테고리에 속한다. (카테고리 정책은 [`../category/01-category-policy.md`](../category/01-category-policy.md) 참조.)
- 브랜드 ID는 필수다. 상품은 반드시 하나의 브랜드에 속한다. (브랜드 정책은 [`../brand/01-brand-policy.md`](../brand/01-brand-policy.md) 참조.)
- 대표 이미지는 선택값이다.
- SKU는 옵션 조합, 가격, 재고를 가진다.
- 주문 항목은 상품이 아니라 SKU를 기준으로 생성된다.
- 상품명과 대표 이미지는 상품 단위의 정보다.
- 옵션, 판매가, 재고는 SKU 단위의 정보다.

예를 들어 "운동화 A"는 상품이고, "운동화 A / 블랙 / 270"은 SKU다.

SKU의 옵션은 다음 규칙을 따른다.

- SKU는 최소 하나 이상의 옵션값을 가져야 한다.
- 옵션명과 옵션값은 비어 있을 수 없다.
- 하나의 SKU 안에서 옵션명은 중복될 수 없다.

## 4. 판매 상태 정책

상품은 다음 판매 상태를 가진다.

| 상태 | 비즈니스 의미 | 고객 노출 | 구매 | 가능한 전이 |
|---|---|---|---|---|
| 판매중 | 정상적으로 판매 중인 상품 | 노출 | 가능 | 판매중지, 단종 |
| 판매중지 | 일시적으로 판매를 멈춘 상품 | 미노출 | 불가 | 판매중, 단종 |
| 단종 | 영구적으로 판매를 종료한 상품 | 미노출 | 불가 | 없음 |

판매중지는 다시 판매중으로 복귀할 수 있다.
단종은 최종 상태이며 다시 판매중으로 복귀할 수 없다.

## 5. 노출 정책

고객 대상 목록, 검색, 상세 화면에는 활성 브랜드의 판매중 상품만 노출한다.

- 판매중 상품은 브랜드가 활성 상태일 때 목록, 검색, 상세 화면에 노출할 수 있다.
- 판매중지 상품은 고객 화면에 노출하지 않는다.
- 단종 상품은 고객 화면에 노출하지 않는다.
- 비활성 브랜드의 상품은 상품 자체가 판매중이어도 고객 화면에 노출하지 않는다.
- 고객이 판매중지 또는 단종 상품의 상세를 요청하면 찾을 수 없는 상품처럼 처리한다.
- 고객이 비활성 브랜드 상품의 상세를 요청해도 찾을 수 없는 상품처럼 처리한다.
- 상품 목록에는 상품별 SKU 판매가 중 최저 판매가를 요약 가격으로 표시한다.

판매중지·단종 상품과 비활성 브랜드 상품을 고객 화면에서 숨기는 이유는 고객이 구매할 수 없는 상품을 탐색·선택하지 않도록 하기 위해서다.

현재 검색은 다음 기준을 사용한다.

- 상품명에 검색어가 포함되는 상품을 찾는다.
- 카테고리 ID가 주어지면 해당 카테고리의 상품을 찾는다. 상위 카테고리(대·중분류)가 주어지면 그 아래 모든 하위 카테고리의 상품까지 포함한다.
- 브랜드 ID가 주어지면 해당 브랜드의 상품만 찾는다.
- 검색 결과에도 활성 브랜드의 판매중 상품만 포함한다.

## 6. 구매 가능 정책

상품 구매 가능 여부는 상품 상태, 브랜드 상태, SKU 재고를 함께 보고 판단한다.

고객은 다음 조건을 모두 만족할 때 SKU를 구매할 수 있다.

- 상품이 판매중 상태다.
- 상품이 속한 브랜드가 활성 상태다.
- SKU가 존재한다.
- SKU의 판매 가능 재고가 주문 수량 이상이다.

상품이 판매중이어도 브랜드가 비활성이거나 특정 SKU의 재고가 부족하면 해당 SKU는 구매할 수 없다.
모든 SKU의 재고가 0이면 상품은 화면에서 품절로 표현될 수 있지만, 상품 상태 자체가 바뀌는 것은 아니다.

## 7. 가격 정책

SKU는 정가와 판매가를 가진다.

- 정가는 할인 전 기준 가격이다.
- 판매가는 고객이 실제 주문·결제에 사용하는 가격이다.
- 신규 SKU는 할인 없이 시작하며 판매가는 정가와 같다.
- 할인 중 여부는 판매가가 정가보다 낮은지로 판단한다.
- 판매가는 정가보다 클 수 없다.
- 가격은 음수가 될 수 없다.
- 할인가를 적용해도 정가는 유지되고 판매가만 변경된다.
- 할인을 해제하면 판매가는 정가로 돌아간다.
- 정가를 변경하면 기존 할인은 초기화되고 판매가도 새 정가와 같아진다.
- 주문은 항상 주문 시점의 판매가를 기준으로 금액을 확정한다.

정가와 판매가를 분리하는 이유는 고객에게 할인 정보를 보여주면서도 주문·결제 계산은 하나의 확정된 판매가로 처리하기 위해서다.

쿠폰 할인은 상품 판매 정책의 범위가 아니며 [`../coupon/01-coupon-policy.md`](../coupon/01-coupon-policy.md)에서 다룬다.
기획전, 회원 등급 할인은 아직 별도 정책으로 분리되지 않았다.

## 8. 재고 정책

commerce 시스템의 SKU 재고는 고객에게 판매 가능한 수량을 의미한다.

- SKU의 판매 가능 재고는 0보다 작을 수 없다.
- 재고가 0인 SKU는 구매할 수 없다.
- 재고를 늘리거나 줄이는 수량은 1 이상이어야 한다.
- 현재 재고보다 많은 수량을 차감할 수 없다.
- 품절은 별도 상품 상태가 아니라 SKU 재고에서 파생되는 표시다.
- 상품 전체 품절은 모든 SKU의 판매 가능 재고가 0일 때 판단할 수 있다.

판매 가능 재고와 창고의 실물 재고는 같은 개념이 아니다.
실물 재고는 WMS가 소유하고, commerce는 고객에게 판매할 수 있는 수량을 소유한다.

## 9. 단종 정책

단종은 상품 판매를 영구적으로 종료하는 결정이다.

- 단종 상품은 고객에게 노출하지 않는다.
- 단종 상품은 구매할 수 없다.
- 단종 상품은 다시 판매중으로 되돌릴 수 없다.
- 단종 처리 후에도 과거 주문 내역에서 참조할 수 있도록 기록은 유지한다.

단종을 삭제가 아니라 상태로 남기는 이유는 과거 주문, 운영 이력, 고객 문의 대응에서 상품 기록이 필요하기 때문이다.

## 10. 예외와 보류 정책

현재 정책에서 의식적으로 보류한 항목은 다음과 같다.

| 항목 | 현재 결정 | 향후 필요성 |
|---|---|---|
| 카테고리 검증 | 해소 | 카테고리 도메인 도입. 등록 시 존재 + 리프(소분류) 검증, 상위 검색 시 하위 펼침. [`../category/01-category-policy.md`](../category/01-category-policy.md) 참조 |
| 브랜드 검증 | 해소 | 브랜드 도메인 도입. 등록 시 존재 검증, 비활성 브랜드 상품은 노출 제외. [`../brand/01-brand-policy.md`](../brand/01-brand-policy.md) 참조 |
| 검색엔진 | 보류 | 상품 수가 늘어나면 검색 품질과 성능 정책이 필요하다 |
| 인증·인가 | 보류 | 상품 등록·수정·단종은 운영자 권한이 필요하다 |
| 상품 기본정보 수정 | 보류 | 판매중 상품의 이름·설명·이미지 변경 정책이 필요하다 |
| 옵션 추가·삭제 | 보류 | 판매 이력이 있는 SKU 변경 가능 범위를 정해야 한다 |
| 기획전·회원 할인 | 보류 | 쿠폰 외 기획전·회원 할인은 별도 정책으로 분리해야 한다 |
| 다중 이미지 | 보류 | 대표 이미지 외 상세 이미지 관리 정책이 필요하다 |
